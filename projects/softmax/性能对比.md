# Softmax 性能对比

## 算子描述

- **计算公式**: `softmax(x) = exp(x - max(x)) / sum(exp(x - max(x)))`
- **输入**: x `[M, N]`
- **输出**: y `[M, N]`
- **数据类型**: float32
- **实现模式**: online softmax 三遍扫描 (max → exp/sum → normalize)，1D grid 每行一个 program

## 瓶颈分析

当前 Triton 实现比 PyTorch 慢 2.7x，主要瓶颈：

1. **1D grid 并行不足**: 每行一个 program，1024 行 = 1024 个 program，远超 48 个 Vector Core，大量调度开销
2. **未使用 XBLOCK/XBLOCK_SUB 模式**: 官方推荐固定核数 + constexpr 循环模式，当前为 GPU 风格大 grid
3. **UB 利用率低**: BLOCK_SIZE=1024 仅占用 1024×4×2=8KB，远未填满 96KB 可用空间
4. **单行处理无法批量化**: 每个 program 仅处理一行，无法利用多行批处理提高向量计算吞吐

## 优化方向

| 优化项 | 预期收益 | 说明 |
|-------|:-------:|------|
| 固定核数 + constexpr 循环 | 高 | 改为 `(num_cores, 1, 1)` grid，消除调度开销 |
| 多行批处理 | 高 | 每个 program 处理多行，提高 Vector 利用率 |
| UB 最优 BLOCK_SIZE | 中 | 填满 96KB doublebuffer 槽位 |
| constexpr r_loops 分块 | 中 | 支持大 num_cols 分块累加 |
