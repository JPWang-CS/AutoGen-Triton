## 5.1 数据类型概览

### 浮点类型

| 类型 | Triton | 字节数 | 范围 | 精度 |
|------|--------|--------|------|------|
| Float32 | `tl.float32` | 4 | ±3.4e38 | ~7 位有效数字 |
| Float16 | `tl.float16` | 2 | ±6.5e4 | ~3 位有效数字 |
| BFloat16 | `tl.bfloat16` | 2 | ±3.4e38 | ~3 位有效数字 |

### 整数类型

| 类型 | Triton | 字节数 | 范围 |
|------|--------|--------|------|
| Int8 | `tl.int8` | 1 | -128 ~ 127 |
| Int32 | `tl.int32` | 4 | ±2.1e9 |
| Int64 | `tl.int64` | 8 | ±9.2e18 |

### NPU 数据类型支持情况

| 操作 | float32 | float16 | bfloat16 | int8 | int32 |
|------|---------|---------|----------|------|-------|
| `tl.load/store` | Y | Y | Y | Y | Y |
| `tl.dot` | Y | Y | Y | Y | N |
| `tl.sum/max/min` | Y | Y | Y | Y | Y |
| `tl.exp/log/sqrt` | Y | Y | Y | N | N |

## 5.2 累加器精度

### tl.dot 的累加精度

`tl.dot` 的输入可以是 float16/bfloat16，但累加器始终使用 float32：

```python
@triton.jit
def matmul_kernel(...):
    # a 和 b 是 float16，但累加结果是 float32
    acc = tl.zeros([BLOCK_M, BLOCK_N], dtype=tl.float32)

    for k in range(0, K, BLOCK_K):
        a = tl.load(...)  # float16
        b = tl.load(...)  # float16
        acc += tl.dot(a, b)  # float32 累加

    # 存储时可选择保持 float32 或转回 float16
    tl.store(c_ptr + ..., acc)              # float32 输出
    tl.store(c_ptr + ..., acc.to(tl.float16))  # float16 输出
```

### 归约操作的精度

对 float16 输入做归约时，建议先转为 float32：

```python
@triton.jit
def safe_sum_kernel(x_ptr, y_ptr, n, BLOCK_SIZE: tl.constexpr):
    pid = tl.program_id(0)
    offsets = pid * BLOCK_SIZE + tl.arange(0, BLOCK_SIZE)
    mask = offsets < n

    # 加载 float16 数据
    x = tl.load(x_ptr + offsets, mask=mask)

    # 转为 float32 再归约，避免精度损失
    x_f32 = x.to(tl.float32)
    total = tl.sum(x_f32, axis=0)

    tl.store(y_ptr + pid, total)
```

## 5.3 类型转换

### .to() 方法

```python
# 在 kernel 内部转换
x_float = x.to(tl.float32)      # 任意类型 -> float32
x_half = x.to(tl.float16)       # 任意类型 -> float16
x_bf16 = x.to(tl.bfloat16)      # 任意类型 -> bfloat16
x_int = x.to(tl.int32)          # 浮点 -> 整数（截断）
```

### 转换时机

```python
@triton.jit
def good_kernel(x_ptr, y_ptr, n, BLOCK_SIZE: tl.constexpr):
    pid = tl.program_id(0)
    offsets = pid * BLOCK_SIZE + tl.arange(0, BLOCK_SIZE)
    mask = offsets < n

    # 加载 float16
    x = tl.load(x_ptr + offsets, mask=mask)

    # ✅ 先转 float32 再计算
    x = x.to(tl.float32)
    result = tl.sqrt(x) + tl.log(x + 1e-8)
    tl.store(y_ptr + offsets, result, mask=mask)
```

## 5.4 NPU 分形对齐

### 什么是分形

NPU 的 Cube Core 执行矩阵乘法时，数据按"分形"（fractal）格式排列。每种数据类型有对应的分形大小：

| 数据类型 | 分形大小 | 说明 |
|---------|---------|------|
| Float16 | 16 x 16 | 16 行 16 列为一个计算单元 |
| BFloat16 | 16 x 16 | 同 Float16 |
| Int8 | 32 x 16 | 32 行 16 列 |

### 对齐要求

`tl.dot` 的输入维度最好是分形大小的整数倍：

```python
# ✅ 推荐：BLOCK_K 是 16 的倍数
BLOCK_K = 128  # 128 / 16 = 8，对齐

# ⚠️ 不推荐：BLOCK_K 不是 16 的倍数
BLOCK_K = 100  # 100 / 16 = 6.25，不对齐，可能有性能损失

# 推荐的 BLOCK_SIZE 配置
recommended_blocks = [16, 32, 64, 128, 256]
```

### UB 对齐

| 存储单元 | 对齐要求 |
|---------|---------|
| UB (Unified Buffer) | 32 字节 |
| L0A / L0B | 512 字节 |
| L0C | 64 字节 |

UB 对齐意味着 BLOCK_SIZE 乘以元素字节数应该是 32 的倍数。对于 float16（2 字节），BLOCK_SIZE 应是 16 的倍数。

## 5.5 数值稳定性

### Softmax 溢出防护

```python
@triton.jit
def stable_softmax(input_ptr, output_ptr, row_stride, n_cols, BLOCK_SIZE: tl.constexpr):
    row_idx = tl.program_id(0)
    row = tl.load(input_ptr + row_idx * row_stride + tl.arange(0, BLOCK_SIZE),
                  mask=tl.arange(0, BLOCK_SIZE) < n_cols,
                  other=float('-inf')).to(tl.float32)

    # ✅ 先减最大值，防止 exp 溢出
    row_max = tl.max(row, axis=0)
    safe_row = row - row_max
    numerator = tl.exp(safe_row)

    # ❌ 不减最大值直接求 exp，大值会溢出
    # numerator = tl.exp(row)  # 危险！

    denominator = tl.sum(numerator, axis=0)
    output = numerator / denominator

    tl.store(output_ptr + row_idx * row_stride + tl.arange(0, BLOCK_SIZE),
             output, mask=tl.arange(0, BLOCK_SIZE) < n_cols)
```

### Layer Norm 的 eps 保护

```python
@triton.jit
def layer_norm_kernel(input_ptr, output_ptr, ..., eps, BLOCK_SIZE: tl.constexpr):
    # 加载并转 float32
    row = tl.load(...).to(tl.float32)

    mean = tl.sum(row, axis=0) / n_cols
    variance = tl.sum((row - mean) * (row - mean), axis=0) / n_cols

    # ✅ 加 eps 防止除零
    rstd = 1.0 / tl.sqrt(variance + eps)  # eps 通常为 1e-5

    normalized = (row - mean) * rstd
    tl.store(..., normalized)
```

## 5.6 dtype 对 BLOCK_SIZE 的约束

BLOCK_SIZE 的选择受 dtype 和 UB 容量（Atlas A2 = 192KB）限制。UB 中所有变量（load 张量、store 张量、中间结果）占用空间之和不能超过 192KB。

### UB 容量计算

```python
_UB_CAPACITY = 192 * 1024  # 192KB，所有变量占用空间之和的上限

def _calc_block_size(element_bytes, num_tensors, multibuffer=True):
    """根据实际张量数计算 UB 安全的 block size

    Args:
        element_bytes: 每个元素的字节数 (如 float32=4, float16=2)
        num_tensors: UB 中同时存在的张量数量
        multibuffer: 是否开启双缓冲 (开启后单槽 96KB)
    """
    ub_per_slot = _UB_CAPACITY // 2 if multibuffer else _UB_CAPACITY
    max_elements = ub_per_slot // (num_tensors * element_bytes)
    return max(1 << (max_elements.bit_length() - 1), 64)

# 示例 (multibuffer=True, 单槽=96KB):
# float32 (4B) + 3 张量: 96KB / 12B = 8192 元素
# float16 (2B) + 3 张量: 96KB / 6B  = 16384 元素
# float32 + 5 张量:       96KB / 20B = 4915 → 4096 元素

# 示例 (multibuffer=False, 全量=192KB):
# float32 (4B) + 3 张量: 192KB / 12B = 16384 元素
# float16 (2B) + 3 张量: 192KB / 6B  = 32768 元素
```

**要点**:
- `num_tensors` 应按实际情况计算：load 的张量 + store 的张量 + 中间变量
- 开启 multibuffer（默认）后 UB 分为双缓冲槽位，单槽 = 96KB
- 编译器中间变量也计入 UB 占用，出现 `ub overflow` 时减小 block size

### GEMM 的 UB 占用

对于矩阵乘法 (tl.dot)，每个分块的 UB 占用：

```python
# A 分块: BLOCK_M * BLOCK_K * sizeof(dtype)
# B 分块: BLOCK_K * BLOCK_N * sizeof(dtype)
# C 分块: BLOCK_M * BLOCK_N * sizeof(dtype)
# 总计: (BLOCK_M * BLOCK_K + BLOCK_K * BLOCK_N + BLOCK_M * BLOCK_N) * sizeof(dtype)
```

**UB 溢出时会报编译错误** `ub overflow, requires xxxx bits while 1572864 bits available!`，需要减小 BLOCK_SIZE 或使用 XBLOCK/XBLOCK_SUB 二级切分。
